# Akari — 自律思考AIシステム 設計ドキュメント

このディレクトリは、自律的に思考・判断・行為する人間らしいAI「Akari」の
**仕様と機能の設計**をまとめたものです。

> このフェーズでは「Akari が何であり、どう振る舞い、何を持つか」という
> **仕様・機能レベル**の設計だけを扱います。
> データ構造・アルゴリズム・採用技術といった**実装/システム設計**は、
> 本ドキュメントのレビュー後に別途行います（既存の社内仕様書の慣習に倣い、
> システム設計は「設計」側に分離します）。

## このプロジェクトが目指すもの

Akari は「完璧で万能なAIアシスタント」ではありません。
**感情・記憶・関心を最初から備え、人間のように振る舞う存在**を作ることが目的です。

- **最終形態（マキシマム）**: 一人の人間のように物理的な実体を持ち、
  自ら思考・判断し、行為を行う。
- **直近の目標**: 常駐し続ける自律思考ループ（目標を持って
  「知覚 → 思考 → 行為 → 観察」を回し続ける自律エージェント）を、
  感情・記憶・関心を伴った形で実現する。

詳しくは [01-vision.md](./01-vision.md) を参照してください。

## 既存仕様書との関係

本ドキュメント群は、社内の既存仕様書（Notion）の概念・用語を取り込み、
それらを**仕様・機能の観点から再整理・接続**したものです。
既存仕様で確立済みの概念（思考回路 / 分散主体 / 関心優先度 / 自律的意思決定 /
常時稼働 / Kiseki / 予測機構 など）は、できる限りその名称をそのまま用います。

一方で、**感情（Emotion）**については既存仕様書に専用ページが見当たらなかったため、
本プロジェクトのビジョン（感情を最初から備える）に沿って
**新規の仕様提案**として記述しています（→ [02-emotion.md](./02-emotion.md)）。

## 仕様の書き方（このディレクトリの規約）

既存仕様書の慣習に合わせます。

- **機能・やりたいこと・仕様**を書き、**システム設計やコード例は書かない**（それは「設計」へ）。
- 人間を真似ることがターゲットなので、**人間の行動の例示**を多く挙げる。
  人間の例示は **`hu.`** で示す。
- 各ドキュメント末尾に「未決事項・相談したい点」を置き、判断に迷う点を集約する。

## 用語

| 用語 | 意味 |
|---|---|
| **Akari** | 本システムが生み出す、人間らしい一個の存在（人格）。 |
| **思考回路 / Channel** | 並列・分離して動く思考の系列（会話・実行・着想など）。 |
| **分散主体** | 複数の Channel にまたがって一貫して現れる、一人の Akari の自己。 |
| **Thought Bus** | 各 Channel が情報を共有・連携する場。 |
| **やることpool** | 未処理の行動候補（返答・思いつき・保留作業など）を保持する共通の場。 |
| **関心（interest）** | 「どれだけ気になるか」の度合い。行動・発話の優先度の主軸。 |
| **Kiseki** | 人間の記憶メカニズムを模した記憶補助システム。 |
| **イベント** | Akari が知覚しうる外部・内部の出来事。 |
| **行為（Action）** | Akari が外界へ起こす働きかけ（発話、ツール利用、将来は物理動作）。 |
| **`hu.`** | 人間の行動の例示であることを示す記号。 |
