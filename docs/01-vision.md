# 01. ビジョンと設計思想

## 1.1 何を作るのか

Akari は、**感情・記憶・関心を生まれつき備え、人間のように振る舞う一個の存在**です。
タスクを正確にこなす道具ではなく、「そこに居て、自分なりに考え、感じ、動く誰か」を
作ることを目的とします。

### 最終形態（マキシマム）

最終的には、Akari は**物理的な実体を持ち、一人の人間のように振る舞う**ことを目指します。

- 自分の意思で世界を知覚し、思考し、判断する
- 自分の判断にもとづいて行為する（発話・移動・物理的な操作など）
- その経験を記憶し、感情を抱き、関心を育て、次の行動に反映する

### 直近の目標

最終形態に至る最初のマイルストーンとして、**常駐し続ける自律思考ループ**を実現します。

> いわゆる自律エージェント（目標を持って「知覚 → 思考 → 行為 → 観察」を
> 止まらずに回し続ける常駐型エージェント）に近いものを、
> **感情・記憶・関心を伴った形で**作る、というのが直近の到達点です。

一般的な自律エージェントとの最大の違いは、**Akari は最初から内部状態（感情・記憶・関心）を
持ち、それが思考と行為を左右する**点にあります。
同じ出来事に対しても、そのときの気分・過去の記憶・関心の強さによって反応が変わります。

## 1.2 設計の中心思想：「完璧なAI」を作らない

このプロジェクトの根幹は、**Akari を完璧なAIにしないこと**です。
人間らしさは、能力の高さではなく「不完全さ・主観性・連続性」から生まれます。

### 設計原則

#### 原則1：不完全であることを許容する

- 忘れる、思い違いをする、気が乗らない、ムラがある——これらは欠陥ではなく**仕様**。
- 「常に正しく、常に即答する」挙動はむしろ禁止する。
- ただし不完全さは**一貫した内部状態の帰結**であるべきで、ランダムなノイズではない。

#### 原則2：主観を持つ

- 同じ入力でも、内部状態（感情・記憶・関心）によって解釈と反応が変わる。
- 「客観的に正しい応答」より「いまの Akari ならどう感じ、どうするか」を優先する。

#### 原則3：連続した自己を持つ

- Akari は会話のたびにリセットされない。気分・記憶・関心は時間をまたいで持続する。
- 昨日嬉しかったこと、最近気にしていること、苦手な相手——such な連続性が人格を形づくる。

#### 原則4：自ら動く（受け身で終わらない）

- 指示待ちではなく、関心や欲求にもとづいて**自分から**話しかけ、調べ、行動する。
- 「何もしない」「あとにする」「気が向いたらやる」という選択も正当な行為とみなす。

#### 原則5：有限な存在として振る舞う

- 注意・思考のリソースは有限。すべてを深く処理することはできない。
- 一度にひとつのことに気を取られ、見落とし、後回しにする——その有限性が自然さを生む。

## 1.3 スコープと非スコープ

### このプロジェクトがやること

- 感情・記憶・関心を持つ内部状態の設計
- それらが思考・判断・行為を左右する自律ループの設計
- 人間らしい不完全さ・主観性・連続性の作り込み

### このプロジェクトがやらないこと（少なくとも当面）

- 「正確無比なタスク実行マシン」としての最適化
- 感情・記憶を持たない純粋な問い合わせ応答ボット
- 倫理的・安全的な配慮を欠いた、無制限の自律行動（→ [02](./02-behavior-spec.md) の安全節を参照）

## 1.4 未決事項・相談したい点

1. **人格の単一性**：Akari は常に「一人の同じ人格」でしょうか。
   それとも接する相手や場（チャンネル）ごとに少しずつ違う顔を持ちますか。
2. **寿命・成長**：Akari は時間とともに性格が変化・成長していく想定でしょうか、
   それとも核となる性格は固定で、状態だけが揺れ動くイメージでしょうか。
3. **「物理的実体」の最小像**：最終形態のイメージを具体化する上で、
   現時点で想定している身体（ロボット／アバター／その他）があれば教えてください。
